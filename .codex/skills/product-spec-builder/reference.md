# Google AI Studio / Gemini 能力清单

本文档是 `product-spec-builder` 在做 AI 优先设计时的参考依据。为每个功能判断"能不能用 AI 做"时，查这份清单。

## 核心能力

### 1. 文本生成与理解

| 维度 | 说明 |
|------|------|
| 用途 | 脚本生成、文案改写、分镜描述、对话生成、摘要提炼 |
| 输入 | 文本 prompt + 可选上下文（角色设定、风格要求、参考文本） |
| 输出 | 结构化或自由格式文本 |
| 限制 | 单次输出 token 上限；长文本需分段生成；创意质量依赖 prompt 精度 |
| 短剧用法 | 脚本初稿生成、分集大纲、角色对话、旁白文案、复盘报告自动撰写 |

### 2. 多模态理解（图片/视频/音频）

| 维度 | 说明 |
|------|------|
| 用途 | 图片内容识别、视频帧分析、音频转文字、画面质量评估 |
| 输入 | 图片/视频/音频文件 + 分析 prompt |
| 输出 | 结构化描述、标签、评分、转写文本 |
| 限制 | 视频长度限制；实时流不支持；细粒度审美判断不稳定 |
| 短剧用法 | 镜头质量初筛、分镜与成片对照检查、素材标签自动打标、配音质量初评 |

### 3. 结构化输出（JSON Mode）

| 维度 | 说明 |
|------|------|
| 用途 | 强制输出 JSON 格式，适合需要程序解析的场景 |
| 输入 | prompt + JSON schema 定义 |
| 输出 | 符合 schema 的 JSON |
| 限制 | schema 复杂度影响输出质量；嵌套过深易出错 |
| 短剧用法 | 试跑单结构化回传、素材标签批量输出、检查报告结构化、成本日志格式化 |

### 4. 函数调用（Function Calling）

| 维度 | 说明 |
|------|------|
| 用途 | AI 根据用户意图自动选择并调用预定义函数 |
| 输入 | 用户指令 + 函数定义列表 |
| 输出 | 函数调用参数 |
| 限制 | 函数定义数量影响选择准确性；复杂参数需明确类型约束 |
| 短剧用法 | 自动路由到对应工作流（试跑、复盘、补库）、自动触发检查、自动分发任务 |

### 5. 系统指令（System Instructions）

| 维度 | 说明 |
|------|------|
| 用途 | 设定 AI 的角色、行为边界、输出格式、交互规则 |
| 输入 | 系统级 prompt（不可被用户覆盖） |
| 输出 | 影响所有后续对话行为 |
| 限制 | 过长的系统指令会压缩可用上下文；规则冲突时行为不可预测 |
| 短剧用法 | 定义 Agent 的指挥官角色、追问规则、输出格式、阶段门判断逻辑 |

### 6. 上下文缓存（Context Caching）

| 维度 | 说明 |
|------|------|
| 用途 | 缓存大量参考材料，降低重复调用成本 |
| 输入 | 需要缓存的文档/数据 |
| 输出 | 缓存 ID，后续调用引用此 ID |
| 限制 | 缓存有 TTL；需要足够的重复调用才划算 |
| 短剧用法 | 缓存素材库标签体系、缓存 playbook 规则集、缓存历史复盘报告 |

### 7. 嵌入向量（Embeddings）

| 维度 | 说明 |
|------|------|
| 用途 | 文本/图片语义检索、相似度匹配 |
| 输入 | 文本或图片 |
| 输出 | 向量表示 |
| 限制 | 需要自建向量数据库；检索质量依赖分块策略 |
| 短剧用法 | 素材库语义检索、相似镜头查找、历史脚本匹配、对标内容搜索 |

### 8. 代码执行（Code Execution）

| 维度 | 说明 |
|------|------|
| 用途 | AI 生成并运行代码来处理数据、计算、文件操作 |
| 输入 | 自然语言任务描述 |
| 输出 | 代码 + 执行结果 |
| 限制 | 沙箱环境限制；不能访问外部网络；执行时间限制 |
| 短剧用法 | 成本计算、ROI 分析、数据汇总、批量文件处理 |

## AI 能力选择决策树

为每个功能做 AI 优先判断时，按以下顺序：

1. 这个任务的输入输出是什么？→ 确定能力类型
2. AI 能做到几分？→ 参考上面的限制列
3. 人工验收点在哪？→ AI 初稿 + 人工验收 vs 全自动
4. 成本是否可接受？→ token / 积分 / 调用次数
5. 能否用结构化输出？→ 优先 JSON Mode 保证可解析

## Prompt 设计要素

当 Spec 中需要为某个 AI 功能写 prompt 时，必须包含：

| 要素 | 说明 |
|------|------|
| 用途 | 这个 prompt 解决什么问题 |
| 输入 | 用户/系统传入什么 |
| 输出 | 期望的输出格式和内容 |
| 完整 prompt | 可直接使用的系统指令或用户指令 |
| 验收标准 | 怎么判断输出质量合格 |
| 降级方案 | AI 输出不合格时的人工兜底路径 |
